# John Locke Essay Skills

[English](README.md) | 简体中文

一套用于撰写 John Locke Institute 征文比赛论文的Skills。它通过分阶段、需要你全程参与的工作流,把往年获奖论文里提炼出的判断方法应用到写作中。

**持续更新中。** History 类目是第一个技能,后续会扩展更多类目,现有技能也会不断打磨。

> 在依赖本技能产出的任何内容之前,请先读「重要声明」一节。请注意，本技能从未经过 John Locke 真实评审的检验,其提炼所依据的论文也仅来自过去的几个年份。

## 功能说明

`john-locke-history-essay` 是 History 类目的协作写作工作流。输入题目和一个粗略的论点,skill把它们推过五个带检查点的阶段——打磨论点、设计论证结构、规划并核实证据、起草、再对照评分标准做审查——并应用从往年获奖作品逆向拆解出的写作手法。它在每个检查点都会停下来,所以主导权始终在你手里。

## 本仓库包含的技能

- **`john-locke-history-essay`** —— History 类目,现已可用。

计划中的类目见「路线图」。

## 目录结构

```
john-locke-history-essay/
├── SKILL.md                  # 工作流:阶段、检查点、护栏
└── references/
    ├── winning-patterns.md   # 从获奖论文提炼的 8 条判断手法
    └── anti-patterns.md      # 用来逐条排查草稿的 12 种失败模式
```

`SKILL.md` 是骨架,参考文件才是实质内容。技能在开始时读取 `winning-patterns.md`,并在审查阶段对照 `anti-patterns.md` 检查草稿。

## 你需要提供什么

本技能**不**把任何一年的比赛规则写死,因为规则每届都在变。每次运行开始时,它会先检查你有没有提供本届规则;如果没有,就停下来向你索取。请提前从 John Locke Institute 官网当届页面复制好以下内容:

- 本届 History 类目的完整题目列表,按官方原文照抄
- 字数上限,以及哪些内容不计入
- 提交要求:文件名格式、引用与注释规则、匿名要求、学术推荐人要求,以及本届的 **AI 使用政策**（请不要**伪造或删除比赛的限制**）
- 评分标准

另外有两样东西需要你自己拿出来:

- 你选定的那一道题
- 你自己的粗略论点

技能负责把你的想法打磨、搭结构、配证据、做压力测试。它不会替你想出论点,也不会替你选题。

## 如何使用

在支持 Agent Skills 的产品里使用。开一段关于 John Locke History 论文的对话,在提示时把当届规则粘进去,再一个检查点一个检查点地往下走。你可以用任何语言和它交流,正文会始终用英文写。

## 重要声明

在信任本技能产出的任何内容之前,请务必读完这一节。

**未经真实评审检验。** 用本技能写出的论文,从未经过 John Locke Institute 或任何人类评审的评估。目前唯一做过的评估是内部进行的:用多个大语言模型,把本技能辅助写出的草稿和往年获奖论文做盲评对比,模型普遍给本技能的草稿打了更高的分。**这并不能说明任何真实情况。**模型的偏好并不能预测人类评审会怎么打分。请把本技能当作起草和自我批评的辅助工具,而不是通往获奖的路径。

**取材样本小且陈旧。** 这些手法逆向拆解自 2019、2020、2021、2023 几届的少数获奖论文。题目风格、评审侧重、乃至规则本身都可能会有变化。当年能赢的,现在未必能赢,所以这些手法只能当作一个起点先验,绝不能当公式来套。

**最终论文归你负责。** 技能会标出未经核实的说法,但你要自己确认每一处来源、事实和引文,自己守住当届字数上限,自己遵守当届提交规则。

**本skill并未在所有模型上实验。** skill产出的论文未在所有模型上做过测试，不同模型的表现可能有差别，请记得检查

**本skill仅供练习研究所用，请勿将本skill产出的论文直接提交到官方比赛。**
